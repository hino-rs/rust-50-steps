# FizzBuzz

## 学習

- Rustは、すべての処理を関数の上に書きます。
`main.rs`の`main`関数は特別で、そこが**プログラムの起点**になります。
- 標準出力には`println!()`(改行あり)や`print!()`(改行なし)を使います。カッコの中には文字列`"text"`を入れます。値を出力させるには、文字列中に`{}`を入れ、第二引数以降に順番通りで渡します。ただし、変数は直接埋め込むこともできます。例えば変数`x`があった場合、`println!("{x}")`として出力が可能です。
- 定数宣言には`const`を使います。定数は**関数の外**で、`const <定数名(アッパースネークケース)>: <型> = <値>`ように定義します。
- このように、特殊に働く文字(`const`,`for`,`if`など)を**キーワード**と言います。これは関数名などにはできません。
- 今回は数値型を扱います。数値型は、**符号付整数型**(`i32`)、**符号無し整数型**(`u32`)、**浮動小数点数型**(`f64`)の3種類があります。異なる型同士で計算はできません。
- 繰り返しは`for`/`while`/`loop`の3つのがあります。
- `for`は`for <変数名> in 範囲 {}` のように使います。
- 範囲は、`n..m`や`n..=m`、`n..`や`..m`、`..=m`や`..`というように表します。これは糖衣構文の一つで、実際には`std::ops::Range`と言った型になります。`n..m`は`n`以上`m`未満、`n..=m`は`n`以上`m`以下です。
- 基本的な条件分岐には`if`を使います。`if <条件> { /* 処理 */ } else if <条件> { /* 処理 */ } else { /* 処理 */ }`のように使います。

## 要件定義

- FizzBuzzを実装してください。
- 開始の数(`START`)と終了の数(`END`)の定数を定義してください。

**FizzBuzzのルール**:

- 開始の値から終了まで、以下のルールに則って出力させます

- 値が`3`で割り切れる場合は`Fizz`
- 値が`5`で割り切れる場合は`Buzz`
- 値が`3`でも`5`でも割り切れる場合は`FizzBuzz`
- それ以外は値をそのまま
