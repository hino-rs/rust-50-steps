# 数当てゲーム

## 学習

- 関数定義には`fn`を使います。`fn <関数名>(引数1: 型1, 引数2: 型2) -> 戻り値の型 {}`のように定義します。もちろん引数や戻り値は任意です。`main`関数のように、引数も戻り値もない関数も作れます。
- 戻り値は`return`で返します。ただし、関数の一番最後の戻り値は、`return`を省略し、`;`を付けないで値を書くだけで返してくれます。以下は例です。

```rust
fn div(a: i32, b: i32) -> i32 {
    if b == 0 {
        return 0;
    } else {
        a / b
    }
}
```

- 変数定義には`let`を使います。関数の上で、`let <変数名>: 型 = 値`のように定義します。**基本的には型を自動で推論してくれるので、型明示は要りません**。
- 変数はデフォルトで不変です。変更しようとするとエラーになります。可変にさせるには`mut`を使います。`let mut <変数名>...`。
- 変数は上書きできます。`let x = 10`と宣言した後、`let mut x = "rust"`のように、変数名はそのままで可変性も型も変えることができます。
- ライブラリを使うには`use`を使います(`use std::cmp::Ordering`など)。
- ランダムな値を作るには、`rand`クレートを使います(導入済み)。`rand::random_range()`というメソッドは、`n..m`といった範囲を引数に渡すことで、範囲内からランダムな値を渡してくれます。
- `loop`は無限ループです。ただ`loop { /* 処理 */ }`のように使います。
- 繰り返しには`continue`や`break`が使えます。他言語と同じです。

### `std::cmp::Ordering`と`match`

- `std::cmp::Ordering`は列挙型というもので、`Ordering::Less`/`Ordering::Equal`/`Ordering::Greater`というバリアントが提供されています。
- `比較元の数値.cmp(&比較対象の数値)`で、`std::cmp::Ordering`が返ってきます。
- `match`を使うと列挙型をエレガントにパターンマッチングできます。以下は例です。

```rust
// 最上部で`use std::cmp::Ordering::*`をしてるとする

let x = 10;
let y = 20;

match x.cmp(&y) {
    Less    => println!("xはyより小さい"), // x < y
    Equal   => println!("xとyは同じ値"),   // x = y
    Greater => println!("xはyより大きい"), // x > y
}
```

`match`は網羅性が必須です。今回、`x.cmp(y)`は`std::cmp::Ordering`なので、`Less`/`Equal`/`Greater`の3つのアームを書きました。どれかが書ければエラーです。

なお、`&y`はの`&`は参照です。今は深く知らなくて大丈夫です。
`cmp`関数が参照型を要求しているので参照型で渡しました。

### 標準入力と型変換

- `std::io::stdin().read_line(&mut String)`でコンソールからの入力を受け付けることができます。
- これは`Result`型と言うものを返します。
- `Result`型も列挙型で、`OK(T)`と`Err(E)`というバリアントを持ちます。
- そのままでは使えないので、中身(`T`や`E`)を取る必要があります。`.expect(エラー時のメッセージ)`メソッドは、中身を取り出す最も簡単な方法です。ですが、もし`Err(E)`だった場合、プログラムが落ちてしまします。
- `String`型の可変変数が必須なので`let mut buf = String::new()`として初期化しておきましょう。
- `read_line()`で取得した文字は、改行(`\n`)などが含まれます。`String`型の`trim()`メソッドは、それらを取り除いてくれます。
- 文字を数値に変換するには`parse()`メソッドが必要です。`parse()`は浮動小数点数にも整数にもなり得ます(ジェネリクス)。なので、欲しい型の明示が必要です。`.parse::<i32>()`とすることで`i32`にしてくれます。`::<>`はターボフィッシュといわれるものです。
- `.parse()`も、成功したか(文字が`"53"`だったなど)か、失敗した(文字が`"rust"`だったなど)かで、`Result`を返します。`.expect()`で取り出しましょう。

## 要件定義

- 数当てゲームを実装してください
- 定数`MIN`と`MAX`を定義し、その間からランダムな値を取得し、ターゲットとしてください。
- 標準入力と数値変換を担う`input`関数を定義してください。
- 当たりでなかった場合、再度入力を促してください。
- `std::cmp::Ordering`を使って判定処理を行ってください。
- ターゲットより入力が小さければ、「もっと大きいです」、その逆は「もっと小さいです」、当たりであれば「正解です!」と出力させてください。
- 正解であればループを抜けてプログラムを終了させてください。
