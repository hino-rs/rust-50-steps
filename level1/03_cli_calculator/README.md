# CLI電卓

## 学習

### 引数

- コマンドライン引数を取得するには`env::args()`を使います
- それを配列として扱いやすくするために、ベクタ型(`Vec<T>`)に変換します

```rust
// use std::env;

env::args().collect::<Vec<String>>(); // String型ベクタ
```

### ベクタ

- ベクタは**構造体**です。出力させるには、`println!("{:?}", v)`か、`println!("{v:?})`というデバッグ形式で記述する必要があります。

ベクタはインデックスで取得できますが、`let x = v[1]`のように取得できないケースがあります。
例えば`v: Vec<i32>`を取得するとします。`v[1]` すると、ベクタの中のインデックス1の値が**自動でコピー**されます。
ですが、例えば`Vec<String>`はコピーされません。ベクタから直接抜こうとしてしまいます。これはエラーになります。
なので、**参照で取得するか、クローン**する必要があります。

- `let x = &v[1]` or `let x = v[1].clone()`

### キャスト

整数を浮動小数点数にしたいケースや、その逆のケースがあります。
`as`を使うことで手軽にキャストできます。

```rust
let mut x = 10; // i32
x = x as f64;   // f64
x = x as u32;   // u32
```

### panic

`expect()`は、`Err`のときにプログラムを強制終了させました。これは内部的に`panic!`させています。
`panic!`はプログラマが呼び出すこともできます。`panic!("メッセージ")`を使うだけです。

`panic!`を使うのは、基本的に**回復不能エラー**のときのみです。
回復不能エラーは、「プログラムを継続すべきではない状態」を指します。

## 要件定義

- CLI電卓アプリを作ってください。
- 浮動小数点数にも対応させてください

以下は実行例です。

```bash
# 入力
cargo run -p cli_calculator -- 50 - 20

# 出力
50 - 20 = 30


# 入力
cargo run -p cli_calculator -- 5.0 / 2.0

# 出力
5.0 / 2.0 = 2.5
```

> ヒント: 今回はパース先を`f64`にしましょう。これで整数と浮動小数点数に対応できます。
