# 文字列リバーサー

## 学習

### 文字列

Rustの文字列には`String`と`&str`(厳密にいうと`str`)の2種類があります。
イメージとしては、`String`は自分の本で、`&str`は借りた本です。
`String`は自分のものなので自由に書き換えることができますし、何より手元にあるので、**自分で本の存在を保証できます**
これがいわゆる **「所有権と借用」** の本質だったりします。

### イテレータ

Rustは関数型プログラミングの要素を取り入れています。
「変数がデフォルトで不変」なのもその一つです。

関数型プログラミングは、

- 基本的に値を直接変えることはしない
- 副作用を避ける
- 関数を値として扱う
- 小さな関数を組み合わせる
- 宣言的に書く

と言った特徴をもちます。
イテレータは「小さな関数を組み合わせる」「宣言的に書く」がもろに出ています。

宣言的は、 **「何をしたいか」** を重視する書き方です。
その対義として、命令型は **「どうやって動かすか」**  を重視する書き方です。

偶数のみを二倍するロジックを比べてみましょう。

**命令型**: `for`で具体的に書く

```rust
let vec = vec![1, 4, 6, 4, 7, 3]; // vec!()でベクタを作れます
let mut result = Vec::new();

for v in vec {
    if v % 2 == 0 {
        let d = v * 2;
        result.push(d);
    }
} 
```

**宣言的**: 関数を駆使して文章のように書く(イテレータ)

```rust
let result = vec.iter()      // ベクタをイテレータ化
    .filter(|n| *n % 2 == 0) // 条件に合うもの(偶数)を抽出
    .map(|n| n * 2)          // 値を変換(2倍)
    .collect::<Vec<i32>>();  // 結果をVec<i32>に集める
```

宣言的であるメリットは多いです。

- `result`を最初からミュータブルにする必要が無い
- パット見で何をしているコードなのかが分かる
- ゼロコスト抽象化: 複雑なロジックになると、宣言的に書いたほうがコードが減ることがあります。ですが、コンパイルによってどう書いても最終的に同じようなコードへ展開されるので、コード量は少なく、パフォーマンス高く実装できます。

最初は命令型が書きやすいですが、早めに宣言的な書き方に慣れるべきです。

### クロージャ

イテレータに使った`map()`や`filter()`は、引数に関数を取ります。関数を定義して渡すのもアリですが、基本的にクロージャを使うべきです。

`|<引数>: <必要に応じて型>| { /* 処理 */ }`と言うように定義します。
引数が必要ないケースも多いです。その場合は引数の部分を`_`にします。

### 文字列操作

イテレータを返すメソッドは多いです。
文字列の`chars()`メソッドもその一つです。
例えば`"Rust"`と言う文字列に使うと、`'R','u','s','t'`というイテレータにしてくれます。
使ってみましょう。

母音をカウントするプログラムです。

```rust
let text = "Rust is an amazing programming language!";
let vowels_count = text.to_lowercase().chars()
    .filter(|c| matches!(c, 'a'|'i'|'u'|'e'|'o')) // matches!()は、第一引数と第二引数を比較し、一致か否かでboolを返します。
    .count();
```

## 要件定義

- 文字列リバーサーを作ってください。
- 文字列をコマンドライン引数で受け取り、それを逆にした文字列を返します。
- 関数定義と`{}`を除いて、1行で実装してください。

**例:**

```bash
# 入力
cargo run -p string_reverser text

# 出力
txet
```

> ヒント: `rev()`を使います。`char`のベクタは`String`に直接変換できます。
